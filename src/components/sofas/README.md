# Product galleries — how they work

All products (every section) live in **one data file**:

- `src/data/products.ts` — owner-edited list of every product
  (name, section, type, price, photos, sold-out, featured, …)
- Plain-language instructions: `PRODUCT_MANAGEMENT_GUIDE.md` (project root)

Developer-facing pieces (the owner never touches these):

- `src/lib/catalog.ts` — normalises the entries (defaults, LQIP merge,
  sold-out visibility), `shareWithPhoto()` (WhatsApp + photo), `formatPrice()`
- `src/lib/categories-registry.tsx` — the five sections: labels, icons,
  curated filter order. Filter buttons are **generated from the data**: a
  button only appears when at least one product uses that `type`, and new
  `type` values get an auto-prettified button automatically.
- `src/components/gallery/product-gallery.tsx` — header + filters + grid + modal
- `src/components/gallery/product-card.tsx` — one reusable card (price,
  old price, condition, Sold Out badge, photo placeholder)
- `src/components/gallery/product-modal.tsx` — details popup
- Icons per section: `sofas/sofa-icons.tsx`, `fridges/fridge-icons.tsx`,
  `ovens/oven-icons.tsx`, `wardrobes/wardrobe-icons.tsx`

Images:

- Existing photos: `public/sofas/`, `public/fridges/`, `public/ovens/`,
  `public/washing-machines/`, `public/wardrobes/`
- New photos: `public/products/` (any folder under `public/` works — the
  path written in `products.ts` just has to match)
- Blur placeholders (`*-lqip.json` in `src/lib/`) are generated by
  `scripts/optimize-images.py` and merged by id in `catalog.ts`; products
  without one simply load without the blur-up effect.
